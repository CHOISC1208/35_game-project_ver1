現在、Cloudinary の画像アセットの準備が整いました。以下のルールに従って、アプリ全体の画像表示ロジックを修正してください。

#### 1. 画像URLの基本ルール

* **Base URL:** `https://res.cloudinary.com/dsuvr4yh8/image/upload/otsukai-timer/`
* **ディレクトリ構造:** `/money/`, `/items/`, `/characters/`, `/ui/`, `/stamps/`
* **ファイル形式:** すべて `.png`（一部アニメーションスタンプは `.gif` の可能性あり）

#### 2. `src/constants/assets.ts` の修正

個別のURLをハードコードするのではなく、以下の関数 `getAssetUrl` を導入し、各アセット定数をこの関数経由で定義するように書き換えてください。

```typescript
const BASE_PATH = "https://res.cloudinary.com/dsuvr4yh8/image/upload/otsukai-timer";

export const getAssetUrl = (category: string, name: string) => `${BASE_PATH}/${category}/${name}.png`;

export const GAME_ASSETS = {
  MONEY: {
    C10: getAssetUrl('money', '10yen'),
    C50: getAssetUrl('money', '50yen'),
    C100: getAssetUrl('money', '100yen'),
    C500: getAssetUrl('money', '500yen'),
    B1000: getAssetUrl('money', '1000yen'),
  },
  // 他のカテゴリ（items, characters, ui, stamps）も同様の形式で定義してください
};

```

#### 3. コンポーネントの修正

以下のコンポーネントを確認し、`placehold.co` のダミー画像ではなく `GAME_ASSETS` または `getAssetUrl` を使用して、Cloudinary から実際の画像を表示するように修正してください。

* `src/components/Register/Coin.tsx`（お金の画像）
* `src/components/Shop/ProductCard.tsx`（商品の画像）
* `src/screens/BriefingScreen.tsx`（店長の画像）
* `src/components/Clock/Clock.tsx`（もし背景画像を使う場合）

#### 4. 注意点

* `missions.ts` などで定義されている商品IDと、Cloudinary上のファイル名が一致するように配慮してください。
* 画像が表示されない場合に備え、`img` タグには適切な `alt` 属性と、読み込み失敗時のフォールバック（または背景色）を設定してください。
