# 📱 アプリ名：おつかいタイマー（Otsukai Timer）

**ターゲット：** 4歳〜8歳の子供、およびその保護者

## 1. プロジェクトの全体像

「お買い物×算数×時計」を組み合わせた知育アプリ。
AI駆動開発（React）に適した設計とし、画像アセットは外部CDN（Cloudinary推奨）から取得することで、アプリの軽量化と柔軟な更新を実現する。

## 2. コア機能（MVP）

### ① おつかいミッション

* **目標設定：** 「○時までに△△を買ってきて！」というキャラクターからの依頼。
* **時計読解：** ターゲット時刻（アナログ時計）と、現在時刻（デジタル/アナログ併記）の照合。
* **商品選択：** 指定された個数の商品を棚から選んで買い物袋に入れる。

### ② レジ・算数システム

* **支払計算：** 合計金額に対し、10円/50円/100円/500円/1000円をドラッグ＆ドロップで投入。
* **お釣り：** （難易度に応じて）お釣りがいくらになるかを確認する。

### ③ コレクション＆報酬

* **スタンプ付与：** クリア後に「変な生き物」や「食べ物」のユニークなスタンプがもらえる。
* **スタンプ帳：** 過去に獲得したスタンプを確認できる画面。

## 3. アセット管理・技術構成

### ① 画像アセット戦略

* **保存先：** `Cloudinary` または `Vercel Blob`（外部ストレージ）。
* **管理方法：** `src/constants/assets.ts` に全ての画像URLを集約管理する。
* **理由：** リポジトリの軽量化、およびアプリを再ビルドせずにスタンプや商品をリモートで追加可能にするため。

### ② フロントエンド

* **Framework:** `React (Next.js / App Router)`
* **Styling:** `Tailwind CSS`（角丸・パステルカラーの知育デザイン）
* **Animation:** `Framer Motion`（お金のドラッグ、スタンプのバウンド演出）

### ③ ディレクトリ構造案

```text
/src
 ├── app                // 画面（ホーム、ゲーム、スタンプ帳）
 ├── components
 │    ├── Clock         // 時計コンポーネント
 │    ├── Register      // レジ・お金操作
 │    └── Shop          // 商品棚・買い物袋
 ├── constants
 │    └── assets.ts     // ★全画像のURL（CDN）をここに定義
 └── hooks              // 算数ロジックやタイマー制御

```

## 4. 画面詳細（UI設計）

1. **Home:** 「おつかいをはじめる」ボタン。
2. **Briefing:** キャラクターがミッションと「制限時間（時計）」を提示。
3. **Market:** 商品をクリックして袋に入れる。
4. **Checkout:** お金を選んでレジに入れる。合計が合えばクリア。
5. **Result:** スタンプ演出 ＋ 「つぎへ」ボタン。

## 5. 収益化・拡張性

* **追加パック：** 新しいステージ（お菓子屋、宇宙等）や限定スタンプをアセットURLの追加のみで実装。
* **保護者向け設定：** 難易度（桁数、時計の分刻み）を調整できるメニュー。
